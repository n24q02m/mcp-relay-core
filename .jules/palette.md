## 2024-05-24 - Accessible Dynamic Forms and Status Alerts
**Learning:** When dynamically generating form fields from a schema, implicit requirements (like `field.required !== false`) are read by screen readers if the `required` attribute is set, but sighted users need a visual cue. Adding an asterisk `*` wrapped in `aria-hidden="true"` satisfies visual users without double-announcing for screen readers. Furthermore, dynamically injected status containers must have `role="alert"` for screen readers to automatically announce the update without the user shifting focus.
**Action:** Always pair HTML5 `required` attributes with an `aria-hidden` visual indicator in dynamic form builders. Always apply `role="alert"` (or `aria-live="assertive/polite"`) to dynamically injected notification containers.
