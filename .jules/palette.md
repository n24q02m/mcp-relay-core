## 2024-05-15 - Dynamic Form Status Announcements
**Learning:** Pure DOM manipulation for status messages in dynamic forms without proper ARIA attributes (`role="status"` and `aria-live="polite"`) fails to alert screen readers when an operation completes (like form submission error or success), severely impacting the accessibility of one-shot setup flows like credential relay. Furthermore, toggle buttons that function like a mode selector should use `aria-pressed` rather than just styling an active class.
**Action:** Always inject dynamic notification and error containers with `role="status"` and `aria-live="polite"` right from their creation, and ensure toggle buttons communicate state using `aria-pressed="true|false"`. Do not rely solely on visual cues like text changes or colors to indicate state or progress.
